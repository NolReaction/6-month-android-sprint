### Task 6 - Мини-аналитический модуль (min/max/avg/topN + суммы по категориям)
**Дано:**  
`data class Purchase(val user: String, val category: String, val amount: Int)`  
и `List<Purchase>` (10–20 элементов для теста)

**Сделать:** реализовать функции:
1) `minAmount()` и `maxAmount()` по списку покупок
2) `avgAmount()` — среднее значение `amount`
3) `totalByCategory()` -> `Map<String, Int>` (сумма `amount` по категориям)
4) `topCategories(n: Int)` -> `List<Pair<String, Int>>` — топ-N категорий по сумме (по убыванию)

**Ожидаемый результат:**
- min/max/avg считаются корректно
- `totalByCategory()` возвращает правильные суммы
- `topCategories(n)` возвращает отсортированный топ-N
