# 30.01.2026 (Week 2 / Day 4) 2~h
## ООП-дизайн: SRP, инкапсуляция, интерфейсы

В этом мини-проекте реализован простой **трекер привычек в консоли** с разделением ответственности между слоями.

---

## Компоненты

- **Habit** - доменная модель (структура данных привычки).
  Содержит `id`, `title` и `doneCount`.

- **HabitRepository** - интерфейс (контракт) для работы с хранилищем привычек.
  Определяет, какие операции доступны: добавить, получить все, найти по id, обновить.

- **InMemoryHabitRepository** - реализация `HabitRepository`,
  которая хранит данные в памяти с помощью `MutableMap`.

- **HabitService** - слой бизнес-логики.
  Отвечает за сценарии:
    - создание привычки
    - отметку выполнения (`doneCount + 1`)
    - получение списка привычек  
      Сервис не печатает данные и не читает ввод.

- **Console UI (main)** - пользовательский интерфейс в консоли.
  Отвечает за:
    - вывод меню
    - чтение ввода пользователя
    - вызов методов `HabitService`
    - вывод результата пользователю
