## Что реализовано

- CLI-приложение с обработкой команд:
    - `add`, `list`, `find <id>`, `start <id>`, `done <id>`, `del <id>`, `stat`, `exit`
- Парсинг пользовательского ввода + aliases команд
- Хранилище задач на `MutableList<Task>`

### Модель данных
- `Task` - immutable data class
- `TaskStatus` - `sealed class`:
    - `NEW`
    - `IN_PROGRESS(startedAt)`
    - `DONE(finishedAt, spentTime)`
    - `REMOVED(removedAt, reason)`

### Работа со временем
- Используется `Instant` и `Duration`
- Засекается время выполнения задачи (`start -> done`)
- Рассчитывается `spentTime`

---

## Ключевые технические моменты, которые понял

- Разница между `List` и `MutableList`
- Почему изменения внутри функции меняют исходный список
- Почему `copy()` + `taskList[index] = updated` - правильный паттерн
- Разница между `is`, `!is` и `as?`
- Безопасные приведения типов (`as?`)
- Почему `Instant` работает в UTC
- Как проектировать состояние (state machine)
